🛠 Development Roadmap

Malaysia dev rates typically 40-60% lower than US/EU

Phase 1 — Validate & Define (Month 1–2)

Duration: 6–8 weeksCost: RM 5,000–15,000
  • Define target niche and primary user persona (optometrist vs patient-facing)
  • Conduct 10–15 user interviews; identify top 3 pain points
  • Competitive audit: map existing solutions and pricing
  • Draft feature list, rank by impact vs effort (MoSCoW)
  • Confirm regulatory classification (Class A–D, MDA Malaysia)
🔒 Gate: Signed user validation report + confirmed regulatory path before proceeding

Phase 2 — MVP Design & Prototype (Month 2–4)

Duration: 6–10 weeksCost: RM 8,000–25,000
  • Wireframe core flows (max 3–5 screens for MVP)
  • Build interactive prototype (Figma or equivalent)
  • 5 usability tests with target users; iterate on findings
  • Define tech stack: React Native / Flutter for mobile; Node.js / Laravel for backend
  • Set up PDPA-compliant data architecture; map personal data flows
🔒 Gate: Prototype usability score ≥70% task success before development begins

Phase 3 — Build MVP (Month 3–7)

Duration: 10–16 weeksCost: RM 20,000–60,000
  • Sprint-based development (2-week sprints); Malaysia dev rates ~RM 80–150/hr
  • Core features only: auth, primary workflow, basic analytics
  • Integrate WhatsApp Business API for notifications (if B2C)
  • PDPA consent flows, data encryption at rest and in transit
  • Internal QA: functional, regression, device compatibility (Android priority in MY)
💰 Funding checkpoint: Apply for MDEC GAIN grant (up to RM 300K) or CRADLE CIP300

Phase 4 — Pilot & Iterate (Month 7–10)

Duration: 8–12 weeksCost: RM 5,000–15,000
  • Recruit 3–5 pilot clinics or 50–100 beta users
  • Instrument analytics: track activation, retention, key actions
  • Weekly feedback loops; fix critical bugs within 48 hours
  • Measure unit economics: CAC, LTV, NPS baseline
  • If Class B+: begin MDA medical device registration process (6–18 months)
🔒 Gate: Retain ≥40% of pilot users at Week 8; NPS ≥30 before public launch

Phase 5 — Launch & Scale (Month 10+)

SEA-first expansionCost: ongoing
  • Public launch on App Store + Google Play; list on AMO platform if applicable
  • Pricing model: freemium → paid tier, or clinic SaaS subscription (RM 100–500/month)
  • Channel partners: AMO, optometry schools (UM, UCSI, MAHSA, KPTM)
  • SEA expansion: Singapore (HSA), Indonesia (BPOM), Thailand after MY traction
  • Series A fundraise or strategic partnership after 12-month data proof

⚠️ Common Pitfalls

Over-building before validation · Skipping PDPA compliance until post-launch · Underpricing B2B SaaS · Ignoring Android-first reality in Malaysia (78% Android market share)

🔍 Research Framework

TAM/SAM/SOM analysis, competitor mapping, user research protocol

📐 Market Sizing Framework

  • TAM (Total Addressable Market): Global optometry digital health market — ~$65.8B (2026). Use WHO prevalence data × average revenue per patient × digitisation rate.
  • SAM (Serviceable Addressable Market): Narrow by geography (SEA/Malaysia), solution category, and tech-adoption segment. Malaysian optometry market ~RM 2.1B.
  • SOM (Serviceable Obtainable Market): Realistic Year 1–3 capture rate. Target 0.5–2% of SAM in early stages. Document assumptions explicitly.
  • Validation: Cross-check with bottom-up calculation (# target clinics × average contract value per clinic).

🔍 Competitor Mapping

  • Direct competitors: Products solving the same problem for the same user (e.g., other DED apps, other tele-optometry platforms)
  • Indirect competitors: Different approach to same job-to-be-done (e.g., WhatsApp for patient reminders vs dedicated app)
  • Map each on: Price · Target market · Key features · Regulatory status · Funding raised
  • Differentiation test: Can you articulate why a user would switch in one sentence? If not, rethink positioning.

👥 User Research Protocol

  • Recruit: 8–12 target users across 2–3 user segments (e.g., optometrist, patient, clinic manager)
  • Method: 30-min semi-structured interviews; observe current workflow before asking about solutions
  • Key questions: What is the most painful part of [workflow]? How do you currently handle it? What would make you pay for a solution?
  • Synthesise: Affinity mapping → identify top 3 themes → validate with survey (n≥50)
  • Malaysian context: Bilingual recruitment (BM + English); account for urban/rural divide in optometry access

✅ Go / No-Go Checklist

  • Problem confirmed by ≥70% of interviewed users as high-priority
  • Willingness-to-pay validated (not just interest — ask for commitment)
  • Regulatory path clear (MDA class, PDPA obligations mapped)
  • SAM ≥ RM 10M (minimum viable market for sustainable business)
  • No single competitor with >60% market share and defensible moat
  • Founding team has domain expertise OR committed clinical advisor
